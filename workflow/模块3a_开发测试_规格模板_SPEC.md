# 模块3a 开发测试_（核心瓶颈模块）_规格模板_SPEC

> 用法：每个新功能复制本模板填写。这是你和AI的唯一事实来源。
> 范例已填：宴请场景的圆桌座位拖拽功能

## 1. 功能目标（一句话）
让用户在宴请场景下，向圆桌拖拽宾客并自动吸附到座位点，完成一桌排布。

## 2. 用户故事
作为【宴会组织者】，我想要【把宾客名字拖到圆桌的座位上】，以便【快速安排谁坐哪】。

## 3. 范围边界（明确不做什么，防止AI过度发挥）
- 本功能只做：单张圆桌的座位拖拽与吸附
- 本功能不做：多桌联动、AI智能推荐排座、导出（留给后续功能）

## 4. 详细行为规格（越具体AI越准）
- 圆桌默认显示 10 个座位点，可在 6–12 间调整
- 从左侧宾客列表拖拽姓名卡片到座位点，松手自动吸附到最近空位
- 座位被占用后变色，宾客卡片从左侧列表移除
- 再次拖拽已落座宾客可换座；拖出桌外则放回左侧列表
- 所有操作即时保存到本地

## 5. 数据结构（让AI先和你确认）
- 桌子: { id, 形状:圆, 座位数, 座位点坐标[] }
- 宾客: { id, 姓名, 所在座位id 或 null }

## 6. 验收标准（DoD的依据，逐条可勾）
- [ ] 能拖拽宾客并吸附到最近空座位
- [ ] 占用座位变色且列表移除该宾客
- [ ] 能换座、能拖回列表
- [ ] 刷新页面后排布仍在（已保存）
- [ ] 有自动化测试覆盖"吸附到最近空位"逻辑

## 7. 技术方案（PLAN，与AI共同确定后填写）
- 首发平台：Web（理由见下方"小白平台建议"）
- 技术栈：React + 拖拽库（如 dnd-kit）+ 浏览器本地存储
- 后续扩展：用 Taro/uni-app 复用逻辑出微信小程序
