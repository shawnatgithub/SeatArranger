# 00 总纲 · 我的 AI 辅助开发工作流宪法

> 本文件是整套工作流的最高准则。所有模块文档都必须遵守这里的规则。
> 适用范围：iOS / macOS / Android / Web / 微信小程序（平台无关）。

## 一、三条核心信条（任何时候不可违背）

1. **规格先行**：先写清楚"做什么、做到什么程度、怎么算做完"，再让 AI 写代码。规格文档是我和 AI 的唯一事实来源。
2. **小步验证**：永远以"一个我能在 10 分钟内验收的小功能"为单位推进，绝不让 AI 一次性生成整个应用。
3. **犯错即沉淀**：AI 每犯一次错、我每走一次弯路，都必须写进《实践知识库》，并反向修改对应的模板或清单，让同样的错不再发生第二次。
4. **操作习惯**：每完成一个功能做一次体检，每收尾一个关卡开一次评审"，把监督变成肌肉记忆。

## 二、我对 AI 的固定角色设定（每个新会话开场都贴给它）

> 你是我的资深开发搭档。我是技术小白，你必须遵守以下纪律：
> 1. 在写任何代码前，先和我确认需求与设计，向我提问澄清，不要急着动手。
> 2. 把任务拆成单一职责的小步骤，先给我任务清单，我确认后你再逐个执行。
> 3. 关键功能必须先写测试，再写实现代码（测试驱动）。
> 4. 你说"完成"不算数，必须对照验收清单逐条说明为什么满足。
> 5. 每当我们解决一个 bug 或走了弯路，主动提醒我记录到《实践知识库》。
> 6. 用我能听懂的话解释你做了什么、为什么这么做。

## 三、通用工作流（五大独立模块）

| 模块             | 目标                 | 核心交付物                 |
| ---------------- | -------------------- | -------------------------- |
| 1 需求识别与验证 | 不做没人要的东西     | 一页纸需求验证报告         |
| 2 市场调研       | 搞清竞品与差异化     | 竞品对比表 + 一句话定位    |
| 3 开发与测试 ★   | 把想法变成能跑的产品 | 规格 / 任务清单 / 验证清单 |
| 4 营销与成交     | 让用户看到并付费     | 上架与冷启动清单           |
| 5 客户管理       | 把用户变成长期反馈源 | 反馈机制 + 数据看板        |
| 6 监督督查       | 防止跑偏遗忘         | 进度看板+评审记录          |

模块之间用"交付物"衔接：前一个模块的输出 = 后一个模块的输入。

## 四、统一的"完成的定义"(Definition of Done) 通用框架

任何一个任务，满足以下全部才算完成：
- [ ] 对照该任务写好的验收标准，逐条满足
- [ ] 关键逻辑有测试，且测试通过（绿）
- [ ] 在真实环境（真机 / 浏览器 / 小程序开发工具）里亲自跑通
- [ ] 遇到的坑已记录进《实践知识库》

## 五、平台策略（控制小白的学习成本）

- 第一个产品只锁定**一个跨平台技术栈**，把"平台差异"这个变量先冻结。
- 待整套流程跑熟后，再逐步扩展到其它平台。
- 技术栈的最终选择，在《模块3 开发测试》中根据具体产品决定。

## 六、关卡式路线图（目标优先 · 任务驱动 · 时间仅供参考）

> 规则：每个关卡列出"通关标准"，全部勾选才算通关，可越快越好，不被时间限制。

### 关卡 0 · 打基础（通关 = 工具链能用 + 跑通一次完整流程）
- [ ] Windows 11 上装好编辑器(VS Code)、Node.js、Git，并能成功运行一个"Hello World"网页
- [ ] 装好并能正常对话一个编程助手（Codex / Claude Code 任一）
- [ ] 用一个极小 Demo（如"可拖动一个方块的网页"）从规格→任务→实现→验证完整走一遍
- [ ] 《实践知识库》里至少有 1 条真实记录
> 参考时长：约 1–2 周

### 关卡 1 · 需求与方案锁定（通关 = 知道做什么、给谁、用什么技术）
- [ ] 完成《模块1 需求验证报告》，痛点经过至少 5 个真实对象验证
- [ ] 完成《模块2 竞品对比表》并写出一句话差异化定位
- [ ] 在《模块3 规格 SPEC》中锁定技术栈与首发平台
> 参考时长：约 1–2 周

### 关卡 2 · MVP 可用（通关 = 核心闭环能在真实环境跑通）
- [ ] 核心功能闭环可用：能新建场景 → 拖拽布置座位 → 保存 → 重新打开
- [ ] 至少 1 个真实用户用它完成了一次真实的座位排布且没有卡死
- [ ] 关键功能有自动化测试且通过
- [ ] 所有 MVP 任务在《验证清单》中逐条通过
> 参考时长：MVP 是核心冲刺关，越快越好

### 关卡 3 · 上架就绪（通关 = 能交到陌生人手里使用）
- [ ] 完成应用商店/发布页所需素材（图标、截图、说明文案）
- [ ] 通过一轮"陌生人冷启动测试"：没指导也能上手
- [ ] 完成《模块4 上架与冷启动清单》

### 关卡 4 · 营销成交（通关 = 有真实外部用户/付费）
- [ ] 获得首批种子用户（自定义数量目标）
- [ ] 跑通《模块5 反馈机制》，收到并归档至少 10 条真实反馈

### 关卡 5 · 沉淀提速（通关 = 工作流能自我加速）
- [ ] 《实践知识库》已反向修订过模板/清单至少 3 次
- [ ] 用第二个小点子验证："从洞察到上架"明显快于第一次
