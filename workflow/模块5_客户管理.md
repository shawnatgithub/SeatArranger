# 模块5_客户管理

> 使命：把一次性用户变成长期反馈源与复购源，并将洞察回流到模块1，闭环工作流。
> 通关标准（对应关卡4）：跑通反馈机制 + 收到并归档至少10条真实反馈。
> 输入：来自模块4的"首批种子用户"
> 本产品范例：座位排布应用

## 一、核心信条
- 你最早的用户是金矿，不是流量数字——他们的吐槽就是产品路线图。
- 收集反馈要"低门槛、有回应"，用户说了有人理，才会持续说。
- 留存比拉新便宜得多，先服务好已有用户。

## 二、反馈收集机制（核心交付物，通关关卡4）
- [ ] 应用内反馈入口（一个"提建议"按钮，一键直达）
- [ ] 建一个用户微信群 / 早鸟群，亲自在里面答疑
- [ ] 评价回复模板：好评致谢 + 差评先共情再说改进计划
- [ ] 反馈归档表（见下），至少积累10条

## 三、反馈归档表
| 序号 | 来源   | 用户原话                 | 类型(bug/需求/夸赞) | 处理状态 | 是否回流模块1 |
| ---- | ------ | ------------------------ | ------------------- | -------- | ------------- |
| 1    | 用户群 | "能不能支持长桌不只圆桌" | 需求                | 待排期   | 是            |
| 2    | 应用内 | "改一个人会不会动到别人" | bug疑虑             | 已解释   | 否            |

## 四、关键数据看板（小白版，先看3个就够）
- **激活**：有多少人完成了第一次成功排座（啊哈时刻）
- **留存**：一周后还回来用的人占比
- **付费/转化**：有多少人触发了付费点
> 不用复杂工具，初期手动统计也行，重点是养成"看数据做决定"的习惯。

## 五、留存与复购动作
- [ ] 新用户首次使用时给一句引导，确保顺利完成第一次排座
- [ ] 对活跃用户主动私聊收集深度反馈（最容易挖出付费点）
- [ ] 重要更新主动通知用户："你提的功能上线了"——这句话极大提升忠诚度

## 六、回流模块1（闭环关键）
> 每月把反馈表里标"是"的需求汇总，作为新的"假设"送回《模块1》验证，
> 决定下一个迭代或下一个产品方向。这就是"洞察→快速上架"的洞察来源。
