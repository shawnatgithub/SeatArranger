# “模拟现场”布局引擎与交互能力评估/重构 Spec

## Why
当前“模拟现场”与编辑画布采用模板静态坐标 + 视图层网格吸附的方式，能满足“可视化 + 手动拖拽微调”，但难以支持更高级的交互：以场地中心为坐标原点的规则化网格、拖拽定位元素后自动重排座椅/桌子、以及稳定可解释的布局规则。

## What Changes
- 评估现有技术架构对以下能力的支持程度，并给出改造路径：
  - 以场地中心点为坐标原点（0,0）的网格系统与布局规则
  - 场地元素可拖拽、拖拽后自动吸附网格
  - 门/窗/主席台/屏幕等定位元素拖拽后，桌子/座椅/座次按规则自动调整
- 选型结论：优先保留 Konva/React-Konva（继续作为渲染与交互层），新增“布局引擎（Layout Engine）”作为领域层能力，替代模板静态坐标的核心地位。
- 将“模拟现场”的布局从“静态坐标集合”升级为“参数化模板 + 约束/规则驱动的可重算布局”：
  - 存储用户可编辑的“定位元素/约束参数”与少量手工 override
  - 在渲染前由布局引擎生成桌子/座椅/辅助图元的最终坐标
- 输出一份规则说明文档 HowSeatsArraged.md（用于解释场地布局与座次排布规则），作为工程交付的一部分。

## Impact
- Affected specs:
  - “模拟现场”布局生成与交互一致性
  - 模板选项与场景策略切换后的可预期渲染结果
  - 定位元素拖拽后自动重排能力（可解释/可复算）
- Affected code:
  - web/src/ui/components/VenueCanvas.tsx（坐标系/网格/吸附/拖拽事件上报）
  - web/src/domain/templates/*（模板从静态坐标向参数化定义迁移）
  - web/src/domain/services/placementService.ts（场景策略与座次安排的规则解释）
  - web/src/app/page.tsx + web/src/ui/components/HomeVenueSimulator.tsx（模板/策略切换后生成正确模拟现场）
  - web/src/app/project/[id]/page.tsx + adapters/storage（持久化：从“逐座位坐标”向“锚点/参数 + override”演进）

## ADDED Requirements
### Requirement: 中心原点网格与规则化布局
系统 SHALL 使用以场地中心为坐标原点（0,0）的世界坐标系（world coordinates）来定义网格与布局规则；渲染层可将 world 坐标映射为画布坐标。

#### Scenario: 网格以中心为 0 点
- **WHEN** 画布渲染网格
- **THEN** 网格线以房间中心点为 0 点对称分布（x 正负、y 正负）
- **AND** 生成的房间与元素位置均以网格系统为基准（生成结果落在网格步长上或遵循明确的对齐规则）

### Requirement: 元素可拖拽与网格吸附
系统 SHALL 支持用户拖拽场地元素与座椅，并在释放时按网格吸附（在 world 坐标空间完成吸附，避免缩放造成误差）。

#### Scenario: 拖拽吸附
- **GIVEN** 用户开启编辑模式
- **WHEN** 用户拖拽任意可拖拽图元并松开
- **THEN** 图元最终位置吸附到最近网格点/线
- **AND** 该位置被持久化并在刷新后保持一致

### Requirement: 定位元素驱动自动重排
系统 SHALL 将门/窗/主席台/屏幕等定位元素作为“锚点（anchors）”；当锚点发生变化时，布局引擎应自动重新计算桌子/座椅等依赖布局，并与座次安排规则协同输出稳定结果。

#### Scenario: 拖拽屏幕后自动重排
- **GIVEN** 场地存在屏幕锚点与桌椅布局
- **WHEN** 用户拖拽屏幕锚点到新位置并释放
- **THEN** 布局引擎重算桌子与座椅布局（例如朝向屏幕、保持对称与间距）
- **AND** 座位 zone 与主位规则保持一致或按模板规则更新

### Requirement: 模板/策略切换可生成正确模拟现场
系统 SHALL 在用户切换“模板选项/场景策略”后，生成与所选项一致的模拟现场与座次安排结果（至少在结构上正确：房间/锚点/桌椅生成正确；策略影响座次分配的偏好与解释）。

#### Scenario: 切换模板
- **WHEN** 用户切换模板并生成模拟现场
- **THEN** 画布呈现该模板对应的布局结构（桌型、座位数量、锚点规则）

#### Scenario: 切换场景策略
- **GIVEN** 存在人员与座位
- **WHEN** 用户切换场景策略并执行排座/预览
- **THEN** 座次安排结果与解释使用新策略名称与偏好逻辑

### Requirement: 规则说明文档
系统 SHALL 在工作区生成 HowSeatsArraged.md，说明：
- 场地布局规则（房间、网格、锚点、桌型生成）
- 座次排布规则（主位、阵营、职级、角色与屏幕偏好等）
